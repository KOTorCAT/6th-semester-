## 1. Цель работы

Изучение проблемы переобучения моделей машинного обучения. Освоение честной процедуры валидации с разделением данных на train/validation/test. Получение практических навыков подбора гиперпараметров через GridSearchCV без использования тестовой выборки.

## 2. Использованные данные

Датасеты из ЛР01:
- cardiovascular_risk.csv (прогноз сердечно-сосудистого риска, 6 признаков)
- credit_risk.csv (прогноз кредитного риска, 6 признаков)

Candidate feature sets из ЛР01:
- full (все признаки)
- set_A (топ-3 признака)

---

## 3. Методология

### 3.1 Разделение данных
- train: 60% данных (обучение и GridSearchCV)
- validation: 20% данных (выбор feature set и сравнение моделей)
- test: 20% данных (финальная проверка, используется один раз)

### 3.2 Процедура
1. Сравнение всех feature sets на train и validation
2. Вычисление generalization gap для выявления переобучения
3. Выбор лучшего feature set для каждой модели по validation F1
4. Построение validation curves для анализа влияния гиперпараметров
5. GridSearchCV на train для подбора оптимальных параметров
6. Финальное сравнение baseline и tuned моделей на test

---

## 4. Результаты

### 4.1 Generalization audit (cardiovascular_risk)

| Feature set | Модель | Train F1 | Val F1 | Gap |
|-------------|--------|----------|--------|-----|
| full | LogisticRegression | 0.58 | 0.52 | 0.06 |
| full | RandomForest | 0.99 | 0.50 | 0.49 |
| set_A | LogisticRegression | 0.56 | 0.53 | 0.03 |
| set_A | RandomForest | 0.99 | 0.51 | 0.48 |

Выводы:
- RandomForest демонстрирует сильное переобучение (gap до 0.49)
- LogisticRegression показывает умеренный generalization gap (0.03-0.06)
- Сокращенный набор признаков немного улучшает validation метрики

### 4.2 Выбор feature set для каждой модели

| Dataset | Модель | Выбранный set | Val F1 | Gap |
|---------|--------|---------------|--------|-----|
| cardiovascular_risk | LogisticRegression | set_A | 0.53 | 0.03 |
| cardiovascular_risk | RandomForest | set_A | 0.51 | 0.48 |
| credit_risk | LogisticRegression | set_A | 0.49 | 0.05 |
| credit_risk | RandomForest | set_A | 0.47 | 0.50 |

Правило выбора: максимизация validation F1, при равенстве минимальный абсолютный gap, предпочтение неполного набора.

### 4.3 Validation curves (LogisticRegression, cardio, set_A)

| C | Train F1 | Val F1 |
|---|----------|--------|
| 0.001 | 0.49 | 0.48 |
| 0.01 | 0.53 | 0.51 |
| 0.1 | 0.56 | 0.53 |
| 1.0 | 0.56 | 0.53 |
| 10.0 | 0.56 | 0.53 |

Вывод: модель стабилизируется при C >= 0.1, дальнейшее увеличение не улучшает качество.

### 4.4 GridSearchCV результаты

LogisticRegression (cardio):
- Лучший C: 0.1
- Mean CV F1: 0.52
- Std CV F1: 0.04

RandomForest (cardio):
- Лучшие параметры: n_estimators=100, max_depth=10
- Mean CV F1: 0.51
- Std CV F1: 0.05

### 4.5 Финальное сравнение на test

| Dataset | Модель | Baseline F1 | Tuned F1 | Baseline AUC | Tuned AUC |
|---------|--------|-------------|----------|--------------|-----------|
| cardiovascular_risk | LR | 0.52 | 0.52 | 0.52 | 0.52 |
| cardiovascular_risk | RF | 0.49 | 0.50 | 0.50 | 0.50 |
| credit_risk | LR | 0.48 | 0.48 | 0.49 | 0.49 |
| credit_risk | RF | 0.46 | 0.46 | 0.47 | 0.47 |

Вывод: на синтетических данных tuning не дает значимого улучшения, что является нормальным результатом.

## 6. Глоссарий

| Термин | Определение |
|--------|-------------|
| Generalization gap | Разница между метриками на обучающей и валидационной выборках |
| Overfitting | Переобучение, когда модель запоминает train вместо выявления закономерностей |
| Validation set | Отложенная выборка для настройки модели без использования test |
| GridSearchCV | Систематический перебор гиперпараметров с кросс-валидацией |
| Validation curve | График зависимости метрики от значения гиперпараметра |
| Baseline model | Модель с параметрами по умолчанию для сравнения |

---

## 7. Выводы

1. RandomForest демонстрирует значительное переобучение на синтетических данных (gap до 0.49), что указывает на необходимость контроля сложности модели

2. LogisticRegression показывает лучшую обобщающую способность с небольшим generalization gap

3. Сокращенный набор признаков set_A дает сравнимые или лучшие результаты по сравнению с полным набором, подтверждая результаты отбора признаков из ЛР01

4. Validation curves показывают, что модель стабилизируется при умеренных значениях регуляризации

5. GridSearchCV не выявил значимого улучшения метрик, что ожидаемо для синтетических данных

6. Честная процедура с отдельным test подтверждает, что tuning не привел к ухудшению результатов

7. Основной результат работы: освоена методика честной валидации, при которой test используется только один раз для финальной проверки
