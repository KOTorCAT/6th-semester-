# Конфигурация SourceCraft для автоматической сборки и деплоя Hugo-сайта

Этот код нужен, чтобы при каждом пуше кода в GitHub автоматически запускалась сборка Hugo-сайта и его выкладка (деплой).

---

## Что делает этот код?

1. **Следит за изменениями** — когда вы пушите код в ветку `main`, запускается процесс.
2. **Проверяет ссылки** — чтобы на сайте не было битых ссылок.
3. **Скачивает и устанавливает Hugo** — нужной версии.
4. **Собирает сайт** — превращает исходники в готовые HTML-файлы.
5. **Деплоит (выкладывает)** — отправляет готовый сайт в отдельную ветку `release`.

---

## Код с подробными комментариями

```yaml
# ==============================================
# БЛОК ТРИГГЕРОВ (КОГДА ЗАПУСКАТЬ)
# ==============================================

on:
  push:                     # Когда происходит пуш (отправка кода в репозиторий)
    workflows: [build-site] # Запускаем workflow (процесс) под названием build-site
    filter:                 # Фильтруем — запускаем только если...
      branches: main        # ...пуш был в ветку main (главная ветка)

# ==============================================
# БЛОК САМИХ ПРОЦЕССОВ (ЧТО ДЕЛАТЬ)
# ==============================================

workflows:
  build-site:               # Название процесса (то же, что указали выше)

    # Переменные окружения — доступны во всех задачах
    env:
      VERSION: 0.157.0      # Версия Hugo, которую будем ставить

    # Список задач (выполняются последовательно, если не указано иное)
    tasks:

      # ---------- ЗАДАЧА 1: ПРОВЕРКА ССЫЛОК ----------
      - name: check-links    # Человеко-понятное имя задачи
        cubes:               # "Кубики" — отдельные шаги внутри задачи
          - name: run-link-check   # Имя шага
            action: tcort/github-action-markdown-link-check@v1  # Используем готовое действие из маркета GitHub
            with:            # Параметры для этого действия
              base-branch: main          # Проверяем ссылки относительно ветки main
              use-verbose-mode: yes      # Выводим подробные отчёты (какие ссылки битые)

      # ---------- ЗАДАЧА 2: СБОРКА И ДЕПЛОЙ ----------
      - name: build-and-deploy
        needs: [check-links]   # Эта задача запустится ТОЛЬКО после успешной проверки ссылок
        
        cubes:

          # КУБ 1: СКАЧАТЬ HUGO
          - name: get-hugo
            script:            # Команды, которые выполнятся в терминале
              - curl -LJO https://github.com/gohugoio/hugo/releases/download/v${VERSION}/hugo_extended_${VERSION}_linux-amd64.deb
              # curl — скачивает файл
              # -L — следовать перенаправлениям
              # -J — сохранить с оригинальным именем
              # -O — сохранить на диск
              # ${VERSION} — подставится 0.157.0 из переменной выше

          # КУБ 2: ПОСМОТРЕТЬ, ЧТО СКАЧАЛОСЬ
          - name: list-files
            script:
              - ls -la        # Вывести список всех файлов с подробностями (права, размер, дата)

          # КУБ 3: УСТАНОВИТЬ HUGO
          - name: install-hugo
            script:
              - sudo dpkg -i hugo_extended_${VERSION}_linux-amd64.deb
              # dpkg -i — установка .deb пакета (как в Ubuntu)
              # sudo — нужны права администратора
              
              - rm hugo_extended_${VERSION}_linux-amd64.deb
              # rm — удаляем установочный файл, чтобы не мусорить

          # КУБ 4: ПРОВЕРИТЬ ВЕРСИЮ HUGO
          - name: check-hugo
            script:
              - hugo version   # Вывести в лог версию Hugo (для отладки)

          # КУБ 5: СОБРАТЬ САЙТ
          - name: build-site
            script:
              - hugo --config hugo.toml --destination ./public
              # hugo — команда для сборки
              # --config hugo.toml — какой файл конфигурации использовать
              # --destination ./public — папка, куда сложить готовый сайт
              
              - ls -la public/   # Показать, что получилось (проверка)

          # КУБ 6: ВЫЛОЖИТЬ САЙТ (ДЕПЛОЙ)
          - name: deploy-site
            script:
              - git checkout -b release
              # Создаём новую ветку release (и переключаемся на неё)
              # В этой ветке будет лежать собранный сайт
              
              - ls -la        # Смотрим, что сейчас в папке
              
              - git add .     # Добавляем ВСЕ файлы в Git (готовим к сохранению)
              
              - "git commit -m \"feat: Deploy Hugo site\""
              # Сохраняем изменения в Git с сообщением "feat: Deploy Hugo site"
              # Кавычки нужны, так как внутри есть пробел
              
              - "git push origin release -f"
              # Отправляем ветку release на GitHub
              # -f (force) — принудительно перезаписываем, если такая ветка уже есть
              # Это полезно для деплоя: каждый раз выкладываем свежую версию
