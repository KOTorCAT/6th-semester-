# Лабораторная работа №2
## Исследование методов оптимизации вычислений

### Цель работы
Исследовать методы оптимизации вычисления кода с использованием потоков, процессов и Cython на примере численного интегрирования методом прямоугольников.

### Установка и запуск

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 setup.py build_ext --inplace
python3 benchmark.py
```

### Результаты измерений

#### Итерация 1: Чистый Python
| n_iter | Время (сек) |
|--------|-------------|
| 10000 | 0.0017 |
| 100000 | 0.0174 |
| 1000000 | 0.1547 |

#### Итерация 2: Потоки (ThreadPoolExecutor)
| n_jobs | Время (сек) | Ускорение |
|--------|-------------|-----------|
| 2 | 0.1653 | 0.94x |
| 4 | 0.1656 | 0.97x |
| 6 | 0.1652 | 0.94x |
| 8 | 0.1494 | 0.98x |

#### Итерация 3: Процессы (ProcessPoolExecutor)
| n_jobs | Время (сек) | Ускорение |
|--------|-------------|-----------|
| 2 | 0.1155 | 1.38x |
| 4 | 0.0584 | 3.29x |
| 6 | 0.0528 | 3.38x |
| 8 | 0.0554 | 3.16x |

#### Итерация 4: Cython
| Метод | n_iter | Время (сек) | Ускорение |
|-------|--------|-------------|-----------|
| Чистый Python | 1000000 | 0.1540 | 1.00x |
| Cython | 1000000 | 0.0250 | 6.15x |

### Выводы

1. Потоки не дают ускорения для CPU-bound задач из-за GIL

2. Процессы дают ускорение до 3.38x на 6 процессах

3. Cython дает максимальное ускорение 6.15x за счет использования C-функций

### Файлы проекта

- `integrate.py` - реализация с doctest, unittest и замерами
- `integrate_parallel.py` - реализации с потоками и процессами
- `integrate_cython.pyx` - Cython версия
- `setup.py` - компиляция Cython
- `benchmark.py` - сравнение всех итераций
- `requirements.txt` - зависимости
