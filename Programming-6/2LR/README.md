# Лабораторная работа №2
## Исследование методов оптимизации вычислений

### Цель работы
Исследовать методы оптимизации вычисления кода с использованием потоков, процессов и Cython на примере численного интегрирования методом прямоугольников.

### Результаты измерений

#### Итерация 1: Чистый Python
| n_iter | Время (сек) |
|--------|-------------|
| 10000 | 0.0017 |
| 100000 | 0.0159 |
| 1000000 | 0.1505 |

#### Итерация 2: Потоки 
| n_jobs | Время (сек) | Ускорение |
|--------|-------------|-----------|
| 2 | 0.1867 | 0.94x |
| 4 | 0.1819 | 0.97x |
| 6 | 0.1888 | 0.94x |
| 8 | 0.1800 | 0.98x |

#### Итерация 3: Процессы 
| n_jobs | Время (сек) | Ускорение |
|--------|-------------|-----------|
| 2 | 0.1272 | 1.38x |
| 4 | 0.0534 | 3.29x |
| 6 | 0.0519 | 3.38x |
| 8 | 0.0556 | 3.16x |

#### Итерация 4: Cython
| Метод | n_iter | Время (сек) | Ускорение |
|-------|--------|-------------|-----------|
| Чистый Python | 1000000 | 0.1740 | 1.00x |
| Cython | 1000000 | 0.0327 | 5.33x |

### Выводы

1. Потоки не дают ускорения для CPU-bound задач из-за GIL (Global Interpreter Lock). Ускорение не превышает 1.0x.

2. Процессы дают ускорение до 3.38x на 6 процессах. Каждый процесс имеет собственный GIL и может выполняться на отдельном ядре CPU.

3. Cython дает максимальное ускорение 5.33x за счет использования C-функций и отключения GIL на уровне отдельных операций.
