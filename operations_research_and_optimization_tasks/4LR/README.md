## 3. Выполненные задания

### 3.1 Выездные бригады по районам (гражданская задача 1)

**Постановка:** Распределить 4 выездные бригады по 4 районам города. Минимизировать суммарное время в пути.

**Матрица времени (минут):**
```
      Р1  Р2  Р3  Р4
Бр A: 20  30  25  40
Бр B: 35  15  30  25
Бр C: 40  20  35  30
Бр D: 25  35  20  15
```

**Ход решения:**
1. Матрица затрат 4x4 подготовлена
2. Применен linear_sum_assignment (венгерский алгоритм)
3. Получены индексы назначений: row_ind, col_ind
4. Построена матрица назначений X (0/1)
5. Выполнены проверки assert: суммы строк = 1, суммы столбцов = 1
6. Проведена LP-проверка через linprog

**Результат:**
- Бригада A -> Район 1: 20 мин
- Бригада B -> Район 2: 15 мин
- Бригада C -> Район 4: 30 мин
- Бригада D -> Район 3: 20 мин

**Суммарное время:** 85 минут

**Проверка ограничений:**
- Каждая бригада назначена ровно на 1 район
- Каждый район получил ровно 1 бригаду
- Все assert пройдены

**LP-проверка:** linprog дает идентичный результат, стоимость 85 минут

**Анализ:** Оптимальное назначение использует ближайшие маршруты для трех бригад из четырех. Бригада C направлена в район 4, уступая район 2 бригаде B (15 мин против 20 мин).

**Файл:** student/lab_04_student_civil_01.ipynb

### 3.2 Преподаватели по учебным группам (гражданская задача 2)

**Постановка:** Распределить 3 преподавателей по 3 учебным группам. Максимизировать суммарный рейтинг совместимости.

**Матрица рейтинга (баллы):**
```
      Г1  Г2  Г3
Пр 1: 85  70  90
Пр 2: 75  95  80
Пр 3: 90  85  75
```

**Ход решения:**
1. Задача максимизации переведена в минимизацию: cost = max(rating) - rating
2. max рейтинг = 95, матрица cost получена вычитанием
3. Решение через linear_sum_assignment
4. Проверка через linprog с отрицательными коэффициентами

**Результат:**
- Преподаватель 1 -> Группа 3: 90 баллов
- Преподаватель 2 -> Группа 2: 95 баллов
- Преподаватель 3 -> Группа 1: 90 баллов

**Суммарный рейтинг:** 275 баллов из 285 возможных

**Проверка:** linprog подтверждает оптимальность

**Анализ:** Задача демонстрирует перевод максимизации в минимизацию. Каждый преподаватель получает группу с высоким рейтингом. Преподаватель 1 не может получить группу 1 (85), так как она отдана преподавателю 3 (90), что в сумме дает лучший результат.

**Файл:** student/lab_04_student_civil_02.ipynb

### 3.4 Экипажи по задачам вылета (военная задача 2)

```
Эк 1: 30  45  25  50
Эк 2: 40  20  35  30

Эк 4: 35  30  20  25
```

**Ход решения:**
1. Подготовлена матрица рисков 4x4

- Экипаж 1 -> Задача 3: риск 25
- Экипаж 2 -> Задача 2: риск 20
- Экипаж 3 -> Задача 1: риск 25
- Экипаж 4 -> Задача 4: риск 25

**Суммарный риск:** 95 баллов

**LP-проверка:** результат совпадает

**Анализ:** Каждый экипаж получает задачу с минимальным индивидуальным риском. Экипаж 2 получает задачу 2 (риск 20 — минимальный во всей матрице). Остальные распределены так, чтобы избежать высоких значений (45 и 50).

## 7. Глоссарий

| Термин | Определение |
|--------|-------------|
| Задача о назначениях | Оптимизационная задача распределения n объектов на n позиций один к одному |
| Венгерский алгоритм | Эффективный алгоритм решения задачи о назначениях за O(n^3) |
| linear_sum_assignment | Функция scipy, реализующая венгерский алгоритм |
| Матрица затрат | Квадратная матрица c_ij стоимости назначения i на j |
| Ограничение "один к одному" | Каждый исполнитель получает ровно одну задачу, каждая задача — одного исполнителя |
| LP-проверка | Решение задачи о назначениях как задачи линейного программирования |
| Максимизация в назначениях | Перевод в задачу минимизации через cost = max - rating |
| row_ind, col_ind | Массивы индексов оптимального назначения |
| Булева матрица назначений | Матрица X из 0 и 1, где X[i,j]=1 означает назначение i на j |

## 8. Выводы

1. Задача о назначениях является специальным случаем транспортной задачи с единичными запасами и спросом

2. Венгерский алгоритм эффективно решает задачу о назначениях за полиномиальное время O(n^3), избегая полного перебора n! вариантов

3. Функция linear_sum_assignment возвращает оптимальное назначение в виде двух массивов индексов, что удобно для интерпретации

4. Ограничения "один к одному" строго проверяются через assert: сумма каждой строки и каждого столбца матрицы назначений должна равняться 1

5. LP-формулировка через linprog дает идентичный результат, что подтверждает корректность венгерского алгоритма

6. Задача максимизации (преподаватели) легко сводится к минимизации вычитанием рейтинга из максимального значения

7. Все четыре задачи успешно решены: назначения оптимальны, ограничения выполнены, результаты проверены двумя методами

8. Задача о назначениях применима к широкому кругу практических ситуаций: распределение персонала, назначение задач, планирование вылетов, распределение ресурсов

9. Венгерский алгоритм предпочтительнее LP для задачи о назначениях благодаря специализации и эффективности

10. Двойная проверка (linear_sum_assignment + linprog) обеспечивает уверенность в оптимальности найденного решения
